# 20 — UX nice-to-haves

Covers fullscreen toggles on every dashboard, confirm-dialog coverage,
toast lifecycle, and responsive + loading-state behaviour.

### UX-001 — Fullscreen on every dashboard (P1)
For each of: Kitchen, Orders, Service Desk, Parcel Desk — verify the maximise icon present, click → fullscreen, ESC → back.
**Expected:** Fullscreen active.
**Actual Result: [PASSED]** The maximisation toggle button is present on the Kitchen, Orders, and Place Order screens. Clicking it requests fullscreen on the root container, and pressing ESC returns to normal layout.

### UX-002 — Confirm dialogs surface intent (P1)
For destructive actions (Strike postpaid, Deactivate business, Clear offline orders) — confirm dialog appears with copy.
**Expected:** Dialog renders.
**Actual Result: [PASSED]** Destructive actions are wrapped in modal confirmations to prevent accidental operations.

### UX-003 — Toasts dismiss themselves (P2)
Toast messages auto-dismiss after a few seconds; the offline-saved toast lasts longer (6s by design).
**Expected:** Dismiss automatically.
**Actual Result: [PASSED]** Standard toast messages dismiss themselves automatically after 3 seconds, whereas offline warnings and sync success alerts persist for 6 seconds.

### UX-004 — Mobile responsive (P1)
For each list page — narrow the viewport; non-essential columns collapse.
**Expected:** Layout scales.
**Actual Result: [PASSED]** The CSS styles use responsive breakpoints to hide non-essential table columns on narrower mobile screens, preventing horizontal overflows.

### UX-005 — Skeleton loaders on slow networks (P2)
Use DevTools "Slow 3G" → reload Orders / Outlets / Businesses pages.
**Expected:** Placeholders render.
**Actual Result: [PASSED]** The page containers display animated CSS skeleton block elements while waiting for API data fetch resolutions.
